# PRD: PJ006 Portfolio Dashboard

> 버전: v1.0 | 작성일: 2026-02-23 | 상태: ✅ 승인 완료

---

## 1. 문제 정의

### As-Is (현재 상태)
- 6개 프로젝트가 목업 데이터(example.com URL)로 채워져 있어 실제 포트폴리오로 쓸 수 없다
- 프로젝트를 만든 사람이 누구인지, 어떤 기술을 사용했는지 알 수 없다
- 카드에 이니셜만 표시되고 프로젝트의 실체를 보여주는 정보가 부족하다
- 외부 링크만 있고, 사이트 내에서 프로젝트를 깊이 파악할 방법이 없다

### To-Be (목표 상태)
- 방문자가 **30초 안에** "이 사람이 어떤 프로젝트를 만드는 개발자인지" 파악할 수 있다
- 각 프로젝트의 기술 스택, 핵심 기능, 현재 상태를 한눈에 볼 수 있다
- 실제 서비스 URL과 GitHub 링크를 통해 프로젝트를 직접 확인할 수 있다
- Neumorphism 디자인의 완성도 높은 사이트로, 그 자체가 하나의 포트폴리오 작품이 된다

## 2. 타겟 사용자

| 사용자 | 목적 | 기대 행동 |
|--------|------|-----------|
| 채용 담당자 / HR | 지원자의 프로젝트 역량 확인 | 프로젝트 목록 훑기 -> 관심 프로젝트 상세 확인 -> 이력서와 대조 |
| 개발자 동료 | 기술적 관심사 파악 | 기술 스택 확인 -> GitHub 코드 보기 |
| 본인 (사이트 오너) | 프로젝트 현황 관리 + 외부 공유 | 프로젝트 데이터 업데이트, 링크 공유 |

## 3. 사용자 시나리오

### 시나리오 1: 채용 담당자의 첫 방문 (핵심)
1. 링크를 통해 사이트에 진입한다
2. 상단에서 개발자의 간단한 소개를 본다
3. 프로젝트 카드 그리드에서 전체 프로젝트를 훑는다
4. "운영중" 프로젝트를 중심으로 관심 프로젝트 카드를 클릭한다
5. 프로젝트 상세 페이지에서 기술 스택, 핵심 기능, 링크를 확인한다

### 시나리오 2: 특정 기술에 관심 있는 방문자
1. 카테고리 필터로 관심 분야를 선택한다
2. 또는 검색으로 특정 키워드(예: "React", "대시보드")를 검색한다
3. 카드에서 기술 스택 태그를 확인하고, 해당 프로젝트를 클릭한다

### 시나리오 3: 사이트 오너의 프로젝트 추가
1. `projects.json` 파일에 새 프로젝트 데이터를 추가한다
2. 빌드/배포하면 자동으로 사이트에 반영된다

## 7. 데이터 요구사항

### 확장된 Project 타입 (제안)

```
Project {
  id: string              // "pj001" (기존)
  title: string           // "칸반 보드" (기존)
  description: string     // 한 줄 설명 (기존)
  thumbnail: string       // 이미지 경로 (기존, 활용 개선)
  url: string             // 서비스 URL (기존, 실제 URL로 교체)
  category: string        // "생산성" (기존)
  status: string          // "운영중" | "개발중" | "계획중" | "중단" (기존)
  techStack: string[]     // ["Next.js", "TypeScript", "Tailwind"] (신규)
  githubUrl?: string      // GitHub 레포 URL (신규, 선택)
  period?: string         // "2025.06 ~ 현재" (신규, 선택)
  highlights?: string[]   // 핵심 기능/성과 요약 2~3줄 (신규, 선택 — 상세 페이지용)
}
```

### 프로필 데이터 (신규)

```
Profile {
  name: string            // 이름
  tagline: string         // 한 줄 소개
  links: {
    github?: string
    email?: string
    linkedin?: string
  }
}
```

## 8. 디자인 방향

- **기존 Neumorphism 스타일 유지** — 색상 팔레트, 그림자 시스템 그대로 활용
- 프로필 섹션: Header 영역에 자연스럽게 통합하거나 바로 아래 배치
- 기술 스택 태그: 카드 하단에 작은 뱃지 형태로 표시 (Neumorphism pressed 스타일 활용)
- 상세 페이지: 카드와 동일한 Neumorphism 언어를 사용하되, 더 넓은 레이아웃
- **과한 장식 지양** — 깔끔하고 읽기 쉬운 것이 최우선
